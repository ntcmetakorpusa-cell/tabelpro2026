# Табель PRO — інструкція з розгортання

Два файли — `index.html` і `app.js` — це повністю готовий застосунок.
Жодного складання чи команд не потрібно.

## 1. Опублікувати на GitHub Pages (відкривається на будь-якому пристрої)

1. Створіть новий репозиторій на GitHub (наприклад, `tabel`).
2. Завантажте у нього **обидва файли**: `index.html` і `app.js`
   (кнопка **Add file → Upload files** → перетягніть → **Commit**).
3. Відкрийте **Settings → Pages**.
   - **Source:** Deploy from a branch
   - **Branch:** `main` / `/ (root)` → **Save**
4. Зачекайте ~1 хв. Зверху з'явиться адреса виду
   `https://ВАШ_ЛОГІН.github.io/tabel/` — це і є посилання на застосунок.
5. Відкрийте його на телефоні, планшеті, ПК — додайте в закладки / на головний екран.

> За замовчуванням дані зберігаються **на кожному пристрої окремо**.
> Щоб бачити їх скрізь одночасно — увімкніть синхронізацію (крок 2).

## 2. Синхронізація між пристроями (безкоштовно, ~5 хв одноразово)

1. Зареєструйтесь на **https://supabase.com** і створіть безкоштовний проєкт.
2. Зліва **SQL Editor → New query**, вставте і натисніть **Run**:

   ```sql
   create table if not exists tp_kv (k text primary key, value jsonb);
   alter table tp_kv enable row level security;
   create policy tp_all on tp_kv for all using (true) with check (true);
   ```

3. **Project Settings → API**, скопіюйте:
   - **Project URL** (виду `https://xxxx.supabase.co`)
   - ключ **anon public**
4. У застосунку натисніть значок 🌥️ (хмара) угорі праворуч → вставте обидва
   значення → **Підключити**. Готово — після цього все, що ви вносите,
   з'являється на всіх пристроях, де відкрито застосунок.
5. Значок 🔄 поряд — оновити дані вручну.

Ключ **anon public** безпечно тримати у застосунку: доступ обмежений
правилами бази, які ви створили запитом вище.

## Перший вхід
Відкрийте застосунок → плитка **«Керуючий»** → придумайте PIN адміністратора.
Далі в меню **«Доступи та PIN»** задайте PIN-коди відділам і кадрам.
